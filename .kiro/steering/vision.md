# Vision

## Problem Statement

従来の仕様駆動開発（Kiro等）では、エージェントが十分なヒアリングを実施せずに仕様を策定していた。
エージェントは空白質問（「何が必要ですか？」）を投げかけるだけで、ユーザの暗黙知や真の要求を引き出せない。
その結果、抽象的な要求のまま仕様化され、手戻りが発生し、良いプロダクトを作ることができない。

## Target Users

- 個人開発者: AIエージェントを活用して高品質な開発を行いたい
- 開発チーム: チーム全体で統一されたAI開発プロセスを導入したい
- 組織: 組織全体でAgentic開発を標準化し、開発効率と品質を向上させたい

規模を問わず、小規模開発から大規模開発まですべてに対応する。

## Project Goal

Q-SPECフレームワークを中心としたAI-DLC（AI Development Life Cycle）全体のベストプラクティスを提供し、
組織の開発者全員がQ-SPEC Kitを導入することで、Agentic開発がスムーズに進む状態を実現する。

具体的には：
- 推測ヒアリングベースの仕様策定により、ユーザの真の要求を引き出す
- AYNiS（ALL YOU NEED iS SPEC）思想により、すべての開発活動をSPECで処理
- SPEC as an Asset思想により、過去のSPECを資産として再利用可能にする
- より高速に、より高品質な開発を実現する

## What We Provide

このプロジェクトは、Amazon Q Developer CLI上で動作するコンテキスト・設定集を提供する：

- Q-SPECフレームワーク定義・ガイドライン
- SPECテンプレート（requirements.md, design.md, tasks.md, notes.md）
- Steering Seeds（設計思想・技術スタックのベストプラクティス集）
- Git運用ルール等の開発プロセス定義
- AI-DLCを実現するための各種コンテキストファイル群

## Core Values

### 柔軟性
プロジェクトごとに最適化可能。
Steeringの仕組みにより、各チームが自分たちのプロジェクトに合わせてカスタマイズできる。
CI/CD、テスト戦略等の実装詳細は、各プロジェクトのsteeringで定義する。

### ベストプラクティスへの即座のアクセス
導入するだけで、ある程度のベストプラクティスに乗れる。
学習コストを最小化し、新メンバーもすぐに高品質な開発プロセスに参加できる。

### 標準化と再利用性
組織全体で統一されたフレームワークを使用することで、知見の共有と再利用が容易になる。
SPECは資産として蓄積され、過去の問題解決プロセスを検索・再利用できる。

## Out of Scope

### エージェント本体の実装
Amazon Q Developer CLIに依存する。
このプロジェクトはエージェントを実装するのではなく、エージェントが読み込むコンテキストを提供する。

### 特定の実装詳細の強制
CI/CD、テスト戦略、デプロイ方法等は、各プロジェクトの実装の問題。
Q-SPEC Kitはフレームワークとガイドラインを提供し、具体的な実装は各プロジェクトのsteeringで定義する。
